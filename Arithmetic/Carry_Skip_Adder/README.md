# Carry Skip Adder

## Purpose
The Carry Skip Adder (CSKA), also known as the Carry Bypass Adder, is an optimized binary adder that reduces carry propagation latency. By detecting if all bit stages in a block will propagate an incoming carry, it can bypass (skip) the serial ripple carry chain for that block.

## Features
- **Block Bypass Multiplexer**: Routes carry signals directly around ripple chains when the propagate condition is met.
- **Uniform 4-bit Partitioning**: Formed using two 4-bit blocks for regular, synthesizable cell layouts.
- **Low latency**: Provides significant delay reduction over Ripple Carry Adders for worst-case additions while maintaining low area overhead.

## Inputs
| Port Name | Bit Width | Type | Description |
|-----------|-----------|------|-------------|
| `a`       | 8         | input| First 8-bit operand vector |
| `b`       | 8         | input| Second 8-bit operand vector |
| `cin`     | 1         | input| Carry-in input |

## Outputs
| Port Name | Bit Width | Type | Description |
|-----------|-----------|------|-------------|
| `sum`     | 8         | output| 8-bit Sum output vector |
| `cout`    | 1         | output| Carry-out output |

## Working Principle
In a typical Ripple Carry Adder, a carry can take up to $N$ gate delays to propagate from LSB to MSB when every stage satisfies $a_i \oplus b_i = 1$ (the propagate condition).

The Carry Skip Adder splits the design into blocks (here, 4-bit blocks) and uses a bypass multiplexer:
1. **Bit Propagate**: Calculates $p_i = a_i \oplus b_i$ for each bit.
2. **Block Propagate**: Calculates $P_{\text{block}} = p_0 \cdot p_1 \cdot p_2 \cdot p_3$. This indicates if every bit in the block is set to propagate a carry.
3. **Carry Bypass**: The block carry-out is computed using a multiplexer:
   $$\text{cout\_block} = (P_{\text{block}}) ? \text{cin\_block} : \text{cout\_ripple}$$
   where $\text{cout\_ripple}$ is the carry-out generated by the normal ripple chain of the block.

### Latency Optimization
In the worst-case carry propagation (e.g. $a = \text{0xAA}, b = \text{0x55}, cin=1$), the carry bypasses the ripple chains of all intermediate blocks, reducing carry-path delay from linear $O(N)$ ripple gates to a small number of multiplexer delay steps.

## Applications
- Energy-efficient arithmetic processors where Carry Lookahead Adders consume too much power.
- DSP logic blocks.
- Embedded controller ALUs.

## Expected Simulation Behaviour
- Sum outputs are correct behavioral addition outputs.
- When `a ^ b == 8'hFF`, changing `cin` from 0 to 1 causes `cout` to toggle with minimal delay, bypassing the internal full adder ripple chains.
